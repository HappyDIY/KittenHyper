构建与开发

构建与开发

本文由 Ranshen_Cervine翻译，原文为 Building and Development

设置环境

  • 支持的平台：
    • Linux x64
    • macOS x64（Intel）
    • macOS arm64（Apple 芯片）
    • Windows x64
  • 仅限Windows：启用开发者模式。这是必需的，因为我们需要符号链接支持。
  • 安装Python 3.8+：
    • 在Unix上，使用您喜欢的包管理器安装python3
    • 在Windows上，从官方网站下载并安装最新版本的Python。
      安装过程中确保选择**“添加Python到PATH”。
    • （Windows上可选）运行pip install colorama安装colorama Python包
  • 安装Git：
    • 在Unix上，使用您喜爱的包管理器安装git
    • 在Windows上，从官方网站下载并安装最新版本的Git。
      安装过程中确保**“启用符号链接”。
  • 安装Android Studio并按照说明完成初次设置。
  • 将环境变量ANDROID_SDK_ROOT设置为Android SDK文件夹路径。这个路径可以在Android Studio设置中找到。
  • 设置JDK：
    • 推荐的选项是将环境变量ANDROID_STUDIO设置为您安装Android Studio的路径。构建脚本将自动找到并使用捆绑的JDK。
    • 您也可以自己设置JDK 17，但本指南不涵盖指令。
  • 克隆源代码：git clone --recurse-submodules https://github.com/topjohnwu/Magisk.git
  • 运行./build.py ndk让脚本为您下载并安装NDK

构建

  • 要构建所有内容并创建最终的Magisk APK，请运行./build.py all。
  • 您也可以构建特定的子组件；调用build.py查看您的选项。
    对于每个操作，使用-h访问帮助（例如./build.py binary -h）
  • 使用config.prop配置构建。提供了一个示例config.prop.sample。

IDE 支持

  • 该仓库可以直接以项目的形式用Android Studio打开。
  • 项目中的Kotlin、Java、C++和C代码应该能够在Android Studio中直接支持。
  • 在处理原生代码前运行./build.py binary，因为一些生成的代码仅在构建过程中创建。

在Android Studio中开发Rust

因为Magisk NDK包，ONDK（使用./build.py ndk安装的那个），包含了一个完全自给自足的Clang + Rust工具链，仅构建Magisk项目不需要配置工具链。然而，由于IntelliJ Rust插件的工作方式，您需要进行一些额外的设置，以使Android Studio能够处理Magisk的Rust代码库：

  • 安装rustup，官方Rust工具链管理器
  • 链接ONDK Rust工具链并将其设置为默认值：
  • 在Android Studio中安装Intellij Rust 插件
  • 在偏好设置 > 语言与框架 > Rust 中，将$ANDROID_SDK_ROOT/ndk/magisk/toolchains/rust/bin设置为工具链位置
  • 打开native/src/Cargo.toml，并在“未找到Cargo项目”横幅中选择“附加”

签名与分发

  • 在发布构建中，签署Magisk APK的密钥的证书将被Magisk的根守护程序用作参考，以拒绝并强制卸载任何不匹配的Magisk应用，以保护用户免受恶意和未验证的Magisk APKs的影响。
  • 要对Magisk本身进行任何开发，请切换到官方调试构建并重新安装 Magisk 以关闭签名检查。
  • 要使用您自己的密钥签名您自己的Magisk构建，请在config.prop中设置您的签名配置。
  • 查看谷歌文档了解更多关于生成您自己的密钥的细节。
