HyperOS 官改内容

HyperOS 官改内容

作者: Ranshen_Cervine ，本文采用CC BY-NC-SA 4.0许可协议，转载请注明出处。

本教程为移植的进阶版本，解锁 HyperOS 功能和获取到相关定制化。涉及到修改的内容会详细说明，请在修改前明确需求，权衡利弊后修改。

system/build.prop

1. 关闭部分log

  1. sys.miui.ndcd=off
    • 功能名称: NDCD（Network Diagnostic Connection Daemon）
    • 功能描述: NDCD 是 MIUI 中的一个网络诊断服务，用于收集网络性能数据，以便于分析和改进网络连接。
    • 影响: 设置为 off 会关闭这项服务，意味着设备不会收集和发送网络性能数据。这有助于保护用户隐私，但也影响小米对网络问题的诊断和改进。
  2. wifi.supplicant_scan_interval=200
    • 功能名称: Wi-Fi 扫描间隔
    • 功能描述: 此设置控制 Wi-Fi 扫描的频率。Wi-Fi 扫描用于发现和连接到附近的 Wi-Fi 网络。
    • 影响: 将扫描间隔设置为200毫秒会使设备更频繁地扫描Wi-Fi网络，提高网络连接的响应速度，但也增加电池消耗。如果设置得更高，比如500毫秒，将减少扫描次数，降低电池消耗，但网络连接的响应速度变慢。
  3. persist.sys.force_sw_gles=0
    • 功能名称: OpenGL ES 软件渲染
    • 功能描述: OpenGL ES 是用于渲染2D和3D图形的软件接口。此设置决定是否强制使用软件（而非硬件）进行图形渲染。
    • 影响: 设置为0使用硬件加速，可以提高图形性能，特别是在游戏和高负载应用中。设置为1强制使用软件渲染，可能减少性能，但在某些特定情况下可以增加兼容性。
  4. persist.wpa_supplicant.debug=false
    • 功能名称: WPA Supplicant 调试
    • 功能描述: WPA Supplicant 是管理Wi-Fi安全的后台服务。此设置控制是否启用调试模式。
    • 影响: 设置为 false 关闭调试模式，减少系统日志的数量，可以提高性能并减少存储空间的使用。如果设置为 true，会生成更多的日志信息，这对于调试网络问题很有用，但可能消耗更多资源。
  5. ro.media.enc.jpeg.quality=80
    • 功能名称: JPEG 压缩质量
    • 功能描述: 此设置决定系统生成 JPEG 图片时的默认压缩质量。
    • 影响: 设置为80意味着图片以80%的质量保存。增加这个值可以提高图片质量，但也会增加文件大小。减少这个值会减小文件大小，但会降低图片质量。
  6. debug.sqlite.journalmode=OFF
    • 功能名称: SQLite 日志模式
    • 功能描述: SQLite 是一个轻量级的数据库系统，用于存储应用数据。日志模式决定了数据库如何记录事务。
    • 影响: 设置为 OFF 关闭日志记录，可以提高数据库操作的性能，但在断电或系统崩溃的情况下可能导致数据库损坏。其他模式如 DELETE、TRUNCATE 或 WAL 可以提供不同级别的数据完整性保护和性能。
  7. miui.usb.dialog=0
    • 功能名称: MIUI USB 对话框
    • 功能描述: 控制当设备连接到 USB 时是否显示对话框。
    • 影响: 设置为0不显示对话框，用户可能不会注意到设备已连接到USB。设置为1会显示对话框，提醒用户选择如何使用USB连接（例如，只充电或文件传输）。
  8. persist.sys.aries.power_profile=low
    • 功能名称: 功耗配置文件
    • 功能描述: 控制设备的功耗模式，以调整性能和电池使用。
    • 影响: 设置为 low 可能会减少设备的性能，以延长电池寿命。如果设置为 high 或其他值，可能会提高性能，但同时消耗更多电池。
  9. ro.kernel.android.checkjni=0 和 ro.kernel.checkjni=0
    • 功能名称: JNI 错误检查
    • 功能描述: JNI（Java Native Interface）允许Java代码与本地代码交互。这些设置控制是否对JNI调用进行额外的错误检查。
    • 影响: 设置为0关闭错误检查，可以提高性能，但也可能导致JNI相关的问题更难发现和调试。在开发和测试阶段，设置为1有助于发现和修正错误。
  10. vendor.bluetooth.startbtlogger=false
    • 功能名称: 蓝牙日志记录
    • 功能描述: 控制是否记录蓝牙相关的日志信息。
    • 影响: 设置为 false 关闭蓝牙日志记录，可以提高性能并减少存储空间的使用。设置为 true 可能有助于调试蓝牙相关的问题，但会增加系统负担。

2. 屏幕帧率

  1. persist.sys.smartfps=0
    • 功能名称: 智能帧率调整
    • 功能描述: 这个设置与系统调整显示帧率以匹配应用需求和节省电池能量的功能有关。
    • 影响: 将其设置为0意味着禁用智能帧率调整功能。这会导致显示帧率不会根据内容动态调整，可能在所有情况下都保持最大帧率，这可以提高视觉流畅度，但可能增加电池消耗。
  2. ro.vendor.fps.switch.default=false
    • 功能名称: 默认帧率切换
    • 功能描述: 这个设置与设备默认的帧率切换策略有关，用于控制在不同场景下是否自动切换帧率。
    • 影响: 设置为 false 可能意味着设备不会默认地自动切换帧率。这会导致设备始终运行在一个固定的帧率下，无论应用需求如何。
  3. ro.surface_flinger.use_content_detection_for_refresh_rate=false
    • 功能名称: 内容检测帧率调整
    • 功能描述: Surface Flinger 是Android中负责组合应用和系统图层以生成最终屏幕图像的组件。这个设置控制Surface Flinger是否根据内容动态调整屏幕刷新率。
    • 影响: 设置为 false 表示Surface Flinger不会基于内容来动态调整刷新率。这可能导致屏幕刷新率不会根据显示内容的需求而变化，可能在所有情况下都保持一个固定的刷新率。
  4. ro.vendor.smart_dfps.enable=false
    • 功能名称: 智能动态帧率调整
    • 功能描述: DFPS（Dynamic Frames Per Second）是动态帧率调整的简称。这个设置与智能动态调整帧率的功能有关。
    • 影响: 设置为 false 表示禁用智能动态帧率调整功能。这可能导致设备不会根据当前的屏幕内容或其他因素动态调整帧率。
  5. ro.vendor.fps.switch.thermal=false
    • 功能名称: 热控帧率切换
    • 功能描述: 这个设置与根据设备的热状态调整帧率有关，用于控制在设备过热时是否降低帧率来减少功耗和热量产生。
    • 影响: 设置为 false 表示不会基于热状态动态调整帧率。这在设备过热时保持高帧率，但可能会导致设备更快过热。
  6. ro.vendor.dfps.enable=false
    • 功能名称: 动态帧率调整
    • 功能描述: 与 ro.vendor.smart_dfps.enable 类似，这个设置也与动态帧率调整有关，但可能涉及不同的系统组件或策略。
    • 影响: 设置为 false 可能表示禁用动态帧率调整功能。这可能导致设备不会根据内容或其他条件动态调整帧率，可能在所有情况下都保持一个固定的帧率。

3. 视频支持

  • 这些设置涉及到设备在音频和视频处理方面的高级功能，包括AI增强、稳定化、帧率转换、后处理和风格化等，需要底层支持。
  1. ro.vendor.audio.aiasst.support=true
    • 功能名称: 音频人工智能助手支持
    • 功能描述: 这个设置指示设备是否支持音频人工智能助手功能，可能涉及语音识别、音频优化或其他智能音频处理功能。
    • 影响: 设置为 true 表示启用音频AI助手。这可能提高音频处理的智能化水平，比如提供更好的噪音抑制、语音识别或音频增强。
  2. ro.vendor.video_box.version=2
    • 功能名称: 视频盒版本
    • 功能描述: 这个设置指定了设备上视频处理软件的版本号。
    • 影响: 版本号通常与特定的功能和性能改进相关联。更高的版本号可能意味着更多的功能和/或性能优化。
  3. debug.config.media.video.aie.support=true
    • 功能名称: 媒体视频人工智能增强支持
    • 功能描述: 类似于音频AI助手，这个设置控制是否启用视频人工智能增强功能，比如通过AI改进视频质量或提供智能视频编辑功能。
    • 影响: 设置为 true 表示启用视频AI增强功能，可能提升视频处理的智能化和用户体验。
  4. debug.config.media.video.ais.support=true
    • 功能名称: 媒体视频AI稳定化支持
    • 功能描述: 这个设置指示设备是否支持利用AI进行视频稳定化处理，以减少摄像过程中的抖动和模糊。
    • 影响: 设置为 true 启用视频AI稳定化功能，可能改善视频拍摄的稳定性和质量。
  5. debug.media.video.frc=false
    • 功能名称: 视频帧率转换（FRC）
    • 功能描述: FRC是一种技术，用于在视频播放时增加帧数，从而使视频播放更加平滑。
    • 影响: 设置为 false 表示禁用FRC。这使得视频播放时不进行额外的帧插入，保持原始帧率。
  6. debug.media.video.vpp=false
    • 功能名称: 视频后处理（VPP）
    • 功能描述: VPP包括一系列用于改善视频播放质量的技术，如去块效应、去噪声等。
    • 影响: 设置为 false 表示禁用VPP。这使得视频播放时不进行额外的图像质量优化处理。
  7. debug.media.video.style=false
    • 功能名称: 视频风格化
    • 功能描述: 这个设置与视频内容的风格化处理有关，比如应用特定的视觉效果或滤镜。
    • 影响: 设置为 false 表示禁用视频风格化功能。这使得视频播放时保持原始风格，不应用额外的视觉效果。
  8. ro.vendor.media.video.frc.support=true
    • 功能名称: 厂商支持的视频帧率转换（FRC）
    • 功能描述: 与上述的 debug.media.video.frc 类似，但这个设置是厂商级别的，表明设备硬件或定制软件是否支持FRC功能。
    • 影响: 设置为 true 表示硬件或厂商定制软件支持FRC，可以提供平滑的视频播放体验。
  9. ro.vendor.media.video.vpp.support=true
    • 功能名称: 厂商支持的视频后处理（VPP）
    • 功能描述: 类似于 debug.media.video.vpp，这个设置表明硬件或厂商定制软件是否支持VPP功能。
    • 影响: 设置为 true 表示支持VPP，可以在视频播放时提供额外的图像质量优化。
  10. ro.vendor.media.video.style.support=false
    • 功能名称: 厂商支持的视频风格化
    • 功能描述: 类似于 debug.media.video.style，这个设置表明硬件或厂商定制软件是否支持视频风格化功能。
    • 影响: 设置为 false 表示不支持视频风格化，视频播放时不会应用额外的风格化处理。

4. 系统相关设置

  • 更改以下设置时需要谨慎，这些设置控制了从图形渲染到网络配置的多个方面，对设备的性能、电池寿命和用户体验有很大的影响。

  1. persist.sys.composition.type=mdp 和 debug.composition.type=mdp

    • 功能名称: 组合器类型设置
    • 功能描述: 这些设置指定系统用于渲染和组合图层的硬件或软件组合器类型。MDP（Mobile Display Processor）是一种硬件组合器。
    • 影响: 设置为MDP意味着系统优先使用硬件组合器进行图像处理和渲染，这可能提高渲染性能和效率。

  2. debug.sf.gpu_comp_tiling=1

    • 功能名称: GPU组合平铺
    • 功能描述: 这个设置与在GPU进行图像组合时使用平铺技术有关，平铺技术可以提高处理效率。
    • 影响: 设置为1启用GPU平铺技术，提高渲染效率。

  3. debug.sf.enable_gl_backpressure=1

    • 功能名称: OpenGL回压控制
    • 功能描述: 这个设置控制系统是否对OpenGL渲染任务施加回压（backpressure），以防止过度渲染和资源浪费。
    • 影响: 设置为1启用回压控制，帮助避免资源浪费并提高性能。

  4. debug.sf.latch_unsignaled=1

    • 功能名称: 未信号化纹理锁定
    • 功能描述: 这个设置与在渲染未完成时锁定纹理相关，防止未准备好的内容被提前渲染。
    • 影响: 设置为1启用这种保护机制，确保只有准备好的内容被渲染。

  5. debug.sf.disable_backpressure=1

    • 功能名称: 禁用回压
    • 功能描述: 与 debug.sf.enable_gl_backpressure 相反，这个设置控制是否禁用OpenGL渲染的回压。
    • 影响: 设置为1禁用回压，会导致更快的渲染速度，但也增加资源消耗。

  6. windowsmgr.max_events_per_sec=1000

    • 功能名称: 窗口管理器事件限制
    • 功能描述: 这个设置限制了窗口管理器每秒可以处理的最大事件数。
    • 影响: 设置较高的值（如1000）可以提高用户界面的响应性，但也增加CPU负载。

  7. debug.sqlite.wal.poolsize=7

    • 功能名称: SQLite WAL连接池大小
    • 功能描述: WAL（Write-Ahead Logging）是SQLite的一种日志模式，提高数据库写入性能。这个设置控制WAL模式下的连接池大小。
    • 影响: 设置更大的池大小可以提高数据库并发写入性能，但也可能增加内存使用。

  8. debug.sqlite.wal.syncmode=OFF 和 debug.sqlite.journalmode=OFF

    • 功能名称: SQLite WAL同步模式和日志模式
    • 功能描述: 这些设置控制SQLite数据库的同步和日志行为。
    • 影响: 关闭同步模式和日志模式可以提高数据库性能，但在系统崩溃的情况下会降低数据一致性。

  9. debug.gralloc.enable_fb_ubwc=1

    • 功能名称: Gralloc UBWC支持
    • 功能描述: Gralloc是Android中用于图像内存分配的库。UBWC（Universal Bandwidth Compression）是一种内存带宽压缩技术。
    • 影响: 启用UBWC可能提高图像处理的效率，减少内存带宽消耗。

  10. persist.vendor.audio.spkr.cal.duration=100

    • 功能名称: 音频扬声器校准时长
    • 功能描述: 这个设置指定了扬声器校准过程的时长。
    • 影响: 设置特定的校准时长可能影响音频质量和扬声器性能。

  11. persist.vendor.camera.imglib.ea_bypass=1

    • 功能名称: 相机图像库早期访问绕过
    • 功能描述: 这个设置与相机图像处理流程中的早期访问（early access）环节有关。
    • 影响: 设置为1可能可以绕过某些图像处理步骤，加快相机响应速度，但也可能影响图像质量。

  12. persist.vendor.camera.llnoise=0

    • 功能名称: 低光噪声抑制
    • 功能描述: 这个设置与相机在低光环境下的噪声抑制功能有关。
    • 影响: 设置为0可能禁用低光噪声抑制，可以在低光环境下提高相机响应速度，但也可能增加噪声。

  13. af.resampler.quality=7

    • 功能名称: 音频重采样质量
    • 功能描述: 这个设置控制音频数据在重采样过程中的处理质量。
    • 影响: 更高的质量设置（如7）可能提高音频质量，但也可能增加处理开销。

  14. ro.data.large_tcp_window_size=true

    • 功能名称: 大TCP窗口尺寸
    • 功能描述: 这个设置控制是否启用更大的TCP窗口尺寸，可以提高网络性能。
    • 影响: 启用大窗口尺寸可以提高网络传输速度，特别是在高延迟的网络上。

  15. pm.sleep_mode=1

    • 功能名称: 睡眠模式
    • 功能描述: 这个设置与设备的睡眠（低功耗）状态配置有关。
    • 影响: 不同的设置值可能影响设备的电源消耗和唤醒行为。

  16. ro.ril.disable.power.collapse=0

    • 功能名称: 禁用电源折叠
    • 功能描述: 这个设置与移动网络硬件的电源管理有关。
    • 影响: 设置为0可能允许电源折叠，有助于节省电能，但也可能影响网络连接的稳定性。

  17. ro.vendor.net.enable_sla=1

    • 功能名称: 网络SLA（服务水平协议）支持
    • 功能描述: 这个设置可能与网络服务的质量保证或优化有关。
    • 影响: 启用SLA支持可能提高网络服务质量，确保更稳定或更高效的网络连接。

  18. persist.sys.allow_sys_app_update=true

    • 功能名称: 允许系统应用更新
    • 功能描述: persist.sys.allow_sys_app_update 这个设置指定了系统是否允许更新预装的系统应用。系统应用通常是与操作系统一起预装在设备上的应用，通常有不同于用户安装的普通应用的更新策略。
    • 影响: 设置为 true 表示系统允许对预装的系统应用进行更新。这意味着，即使应用是作为操作系统的一部分预装的，也可以通过正常的应用更新机制接收和安装更新。这可以确保系统应用能够接收到安全更新、性能改进和新功能，但也可能带来更频繁的更新和潜在的系统稳定性风险。如果设置为 false，则系统应用将不会接收更新，这可能会提高系统的稳定性，但同时也意味着系统应用可能无法及时接收重要的更新和修复。

  19. debug.sf.hw=1

    • 功能名称: 硬件加速渲染

    • 功能描述: debug.sf.hw 是一个用于控制 Android 的 SurfaceFlinger 服务的设置，SurfaceFlinger 负责合成屏幕上显示的所有视图和动画。当这个设置为 1 时，它会启用硬件加速渲染，意味着 SurfaceFlinger 将尽可能使用 GPU（图形处理单元）来渲染界面元素和动画，而不是使用 CPU。

    • 影响: 设置为 1 表明启用硬件加速。这通常可以提高 UI 渲染的性能和流畅度，因为 GPU 在处理图形任务时比 CPU 更高效。这可以使界面动画更加平滑，响应更快，提高整体的用户体验。但是，它也可能增加 GPU 的功耗，特别是在渲染复杂界面或运行图形密集型应用时。

  20. ro.telephony.call_ring.delay=0

    • 功能名称: 电话呼入铃声延迟
    • 功能描述: ro.telephony.call_ring.delay 设置用于控制设备在收到呼入电话时，铃声响起之前的延迟时间。这个值的单位通常是毫秒。
    • 影响: 设置为 0 意味着没有延迟，即电话一旦收到呼入请求，铃声立即响起。这可以确保用户能够尽快得知有呼入电话。有时候，为了同步铃声与震动或者确保不立即打断用户，系统或用户可能会设置一个小的延迟。但在这个设置为 0 的情况下，目标是最快地提醒用户有呼入电话。

  21. windowsmgr.max_events_per_sec=150

    • 功能名称: 窗口管理器最大事件处理率
    • 功能描述: windowsmgr.max_events_per_sec 设置用于控制窗口管理器每秒可以处理的最大事件数。事件可以是触摸、滑动、按键等用户输入事件。
    • 影响: 设置为 150 表示窗口管理器每秒最多可以处理 150 个事件。这个设置影响用户界面的响应性。较高的值可以提高界面的反应速度，使得滑动和触摸等操作感觉更加流畅，但如果设置得过高，可能会对系统性能产生负面影响，特别是在资源有限的设备上。相反，较低的值可能会使界面感觉不够灵敏。

  22. debug.performance.tuning=1

    • 功能名称: 性能调优调试
    • 功能描述: 此设置可能与启用系统级的性能调优相关。具体细节可能因设备和操作系统版本不同而有所差异，但通常，开启此选项会激活一些调试选项或特性，旨在优化设备性能。
    • 影响: 设置为 1 表明性能调优调试功能被激活。这可能导致系统调整某些参数或行为以提升性能，例如提高 CPU 或 GPU 的频率、调整内存管理策略等。然而，这也可能导致增加电量消耗或降低设备的稳定性。通常，这个选项被开发者用于测试和优化应用。

  23. video.accelerate.hw=1

    • 功能名称: 视频硬件加速
    • 功能描述: 此设置指示系统在播放视频时是否启用硬件加速。硬件加速是指使用专门的硬件（如 GPU）来处理视频解码和播放，而不完全依赖于 CPU。
    • 影响: 设置为 1 表示启用视频硬件加速。这通常可以显著提高视频播放的性能和效率，减少卡顿和延迟，同时降低 CPU 负载，从而可能减少电量消耗。然而，某些视频格式或编解码器可能不支持硬件加速，或者在特定硬件上的支持有限。

8. TCP/IP 网络堆栈配置

  1. net.ipv4.tcp_ecn=0
    • 功能名称: TCP 显式拥塞通知 (ECN)
    • 功能描述: ECN 允许网络节点在发生拥塞之前通知发送者减少其传输速率。
    • 影响: 设置为0禁用 ECN。这意味着即使网络设备支持 ECN，TCP 连接也不会尝试使用它来避免拥塞。
  2. net.ipv4.route.flush=1
    • 功能名称: 路由表刷新
    • 功能描述: 控制 Linux 内核路由表的刷新操作。
    • 影响: 设置为1会清除路由缓存。这通常用于网络配置更改后，确保数据包根据最新的路由信息发送。
  3. net.ipv4.tcp_rfc1337=1
    • 功能名称: 遵守 RFC 1337
    • 功能描述: RFC 1337 是一个关于 TCP TIME-WAIT 状态处理的标准。
    • 影响: 设置为1表示遵守 RFC 1337，防止 “TIME-WAIT” 攻击，提高系统稳定性。
  4. net.ipv4.ip_no_pmtu_disc=0
    • 功能名称: 禁用路径 MTU 发现
    • 功能描述: 路径 MTU 发现用于确定最优的数据包大小，避免在传输路径中发生分片。
    • 影响: 设置为0启用路径 MTU 发现，有助于优化数据包大小，提高网络效率。
  5. net.ipv4.tcp_sack=1
    • 功能名称: TCP 选择确认 (SACK)
    • 功能描述: SACK 允许接收者告知发送者哪些数据被成功接收，有助于高效处理丢包情况。
    • 影响: 设置为1启用 SACK，可以提高在丢包环境下的 TCP 性能。
  6. net.ipv4.tcp_fack=1
    • 功能名称: TCP 前向确认 (FACK)
    • 功能描述: FACK 是一种改进的 SACK 机制，用于更高效的丢包恢复。
    • 影响: 设置为1启用 FACK，可以进一步提高丢包恢复的效率。
  7. net.ipv4.tcp_window_scaling=1
    • 功能名称: TCP 窗口缩放
    • 功能描述: 窗口缩放用于支持大于 64KB 的 TCP 窗口，有助于提高高延迟和高带宽网络的性能。
    • 影响: 设置为1启用窗口缩放，允许 TCP 窗口大小超过 64KB，提高在高速网络上的传输性能。
  8. net.ipv4.tcp_timestamps=1
    • 功能名称: TCP 时间戳
    • 功能描述: 时间戳用于更精确的 RTT (往返时间) 测量和避免旧的数据包干扰连接。
    • 影响: 设置为1启用时间戳，可以提高 TCP 性能和可靠性。
  9. net.ipv4.tcp_rmem 和 net.ipv4.tcp_wmem
    • 功能名称: TCP 读/写缓冲区大小
    • 功能描述: 这些设置控制 TCP 读缓冲区和写缓冲区的最小、默认和最大大小（以字节为单位）。
    • 影响: 适当调整这些值可以优化 TCP 性能，特别是在特定的网络条件下。
  10. net.ipv4.tcp_mem
    • 功能名称: TCP 内存缓冲区大小
    • 功能描述: 控制 TCP 堆栈可以使用的内存总量。
    • 影响: 设置合理的值可以优化系统资源使用和网络性能。
  11. net.ipv4.tcp_no_metrics_save=1
    • 功能名称: 禁止 TCP 度量保存
    • 功能描述: 此设置控制是否在 TCP 连接关闭后保存关于连接的某些度量。
    • 影响: 设置为1禁止保存度量，可以在某些情况下节省资源，但可能导致连接建立时缺乏优化信息。

  1. persist.sys.background_blur_supported=true
    • 功能名称: 背景模糊支持
    • 功能描述: 指示设备是否支持 UI 中的背景模糊效果。
    • 影响: 设置为 true 表明设备支持在用户界面中使用背景模糊效果，如通知面板、应用抽屉背景等。
  2. persist.sys.background_blur_status_default=true
    • 功能名称: 默认背景模糊状态
    • 功能描述: 控制默认情况下是否启用背景模糊效果。
    • 影响: 设置为 true 表明背景模糊效果默认是启用的。
  3. persist.sys.background_blur_mode=0
    • 功能名称: 背景模糊模式
    • 功能描述: 指定使用的背景模糊效果的模式或类型。
    • 影响: 不同的值可能代表不同的模糊算法或强度。
  4. persist.sys.background_blur_version=2
    • 功能名称: 背景模糊版本
    • 功能描述: 指示背景模糊功能的版本或实现。
    • 影响: 版本不同可能意味着模糊效果的质量、性能或算法有所不同。
  5. debug.game.video.speed=true 和 debug.game.video.support=true
    • 功能名称: 游戏视频调试
    • 功能描述: 这些设置可能与游戏中的视频播放性能优化有关。
    • 影响: 设置为 true 可能启用了针对游戏视频播放的特定调试或优化，以提高游戏中视频内容的播放速度和质量。
  6. persist.sys.support_view_smoothcorner=true 和 persist.sys.support_window_smoothcorner=true
    • 功能名称: 支持视图和窗口的平滑角
    • 功能描述: 这些设置指示系统是否支持在 UI 视图和窗口边缘显示平滑的圆角。
    • 影响: 设置为 true 表明系统支持在视图和窗口边缘呈现平滑圆角，这可能是为了改善视觉美感和一致性。
